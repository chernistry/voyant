✈️ VOYANT Travel Agent CLI
───────────────────────────────────────
Ask travel questions. I answer in concise English.
You can ask in any language; I will reply in English.
───────────────────────────────────────
What I can help with:
• Weather — "Weather in London this week?"
• Packing — "What to pack for Tokyo in March?"
• Attractions — "Kid‑friendly things to do in SF in late Aug"
• Destinations — "Where to go from Tel Aviv in August?"
• Flights — "Flights from NYC to London March 15" (live search via Amadeus)
• Policies (RAG) — "United baggage allowance", "Marriott cancellation"
and visas — "Do I need a visa for Japan with a US passport?"
• Web search on consent — events, flights, live info
───────────────────────────────────────
How to ask:
• Include city and month/dates when possible.
• I may ask to use web search or deep research — reply "yes" to proceed.
• I avoid prices/budgeting; I can still suggest options.
───────────────────────────────────────
Commands:
/why Show how I got my answer (sources, reasoning, fact-checking)
exit Quit

Key Features

shield

Anti‑Hallucination

Self‑evaluates to prevent hallucinations, requiring citations and secondary verification.

travel_explore

Web Insights

Crawlee/Playwright crawling with anti-bot stealth for policy extraction. Brave/Tavily search with fallback when internal data insufficient.

psychology

Smart Routing

Transformers multilingual NLP with zero-shot routing. 20-40% LLM reduction via Regex→Transformers→LLM cascade. Natural multi-turn support.

public

Multi-Source Data Integration

Connects to Open‑Meteo, REST Countries, OpenTripMap, and Amadeus for verified travel info.

search

Enterprise-grade RAG

Advanced retrieval-augmented generation with Vectara integration for searching and retrieving information from internal knowledge bases, policy documents, and enterprise data sources.

bolt

Clean Architecture

CLI/REST API with TypeScript validation, 685+ tests, circuit breakers, rate limiting, and Prometheus observability.

Quick Start

# Clone repository and enter root directory
git clone https://github.com/chernistry/voyant/ && cd voyant/root

# Install dependencies
npm install

# Start CLI
npm run cli

# Or start HTTP server
npm run dev

# Run tests
npm test

Notes on Prompt Engineering

  • Format Priming: Enforced JSON/bullet-point schemas ensure structured, parseable outputs while preventing hallucinations.
  • Few-Shot Prompting: Curated examples in router.md and cot.md train accurate intent classification and slot extraction.
  • Chain-of-Thought: Structured reasoning in cot.md and verify.md decomposes analysis into verifiable steps.
  • Guardrails: blend.md requires fact-grounding + verify.md provides secondary validation of claims.
  • Query Optimization: Rewrites user search queries to reduce noise.

Other Perks

  • Anti-hallucination receipts with self-check and sources.
  • Resilient data blend: APIs → search fallback when providers fail.
  • Smart routing + memory for natural multi-turn conversations.
  • Optional /metrics endpoint (Prometheus or JSON) and structured logs (PII‑redacted by default).
  • Security-conscious: host allowlist, timeouts, retries with jitter, and circuit-like backoff.
  • E2E transcripts and golden tests for the core scenarios.
